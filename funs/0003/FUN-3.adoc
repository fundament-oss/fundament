= FUN-3 Organizational hierarchies

* Organizations
* Clusters
** A physical bare-metal kubernetes cluster
* Projects
** Multiple projects can be used to setup separate environments (test/staging/prod), since users are granted permissions per project this also allows for isolation of users per environment.
* Namespaces
** Sit below projects, to separate units of deployment within a project
* Users
** Are granted permissions in projects
