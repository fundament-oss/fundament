= FUN-4 Organizational hierarchies

* Organizations
* Clusters
** A physical bare-metal kubernetes cluster
* Projects
** Multiple projects can be used to setup separate environments (test/staging/prod)
** Users are granted permissions per project, allowing for isolation of users per environment.
** A project is linked to one cluster, on which all its resources are deployed.
* Namespaces
** Sit below projects, to separate units of deployment within a project.
* Users
** Are granted permissions in projects.
