## Deviations from PLAN.md (decided, not open — recorded for the reviewer)

- [x] **Step 7 (nginx font CORS header) skipped — not needed.** Bun 1.3.6's CSS
      bundler inlines the woff2 fonts as base64 `data:` URIs (verified: even
      `loader: { '.woff2': 'file' }` still inlines), so the opaque-origin iframe makes
      **no cross-origin `@font-face` fetch**. Adding `Access-Control-Allow-Origin: *`
      to `location /plugin-ui/` would only loosen security for no gain. `nginx.conf`
      is unchanged. If a future bundler change emits separate font files, re-introduce
      Step 7.
- [x] **No `public/plugin-ui/fonts/` directory** (Steps 2–3). Fonts live inline in
      `nldd.css` (~700 KB). The preview serves only `nldd.{js,css}`.
- [x] **Submit is a `click` handler, not `type="submit"`.** Note: in 0.8.53
      `nldd-button` *is* form-associated (`type="submit"` → `requestSubmit()`), but the
      manual click+validate path is kept to avoid double-submission and because
      validation is manual anyway.
